inventor.FieldContainer

class FieldContainer(object)

Base class for scene objects of type SoFieldContainer.

All field values and node kit parts are dynamically added as class attributes. Please refer to the Open Inventor documentation for the fields of each scene object type.

Methods defined here:

check_type(...)

Checks if a scene object is derived form a given type.

Args:

    Name of type to check for as string.

Returns:

    True if the instance is derived from the given type.

connect(...)

Connects a field from another field or engine output.

Args:

  • toField: Field name of this instance that will be conntected.
  • fromObject: Scene objects whose output or field serve as input.
  • fromField: Field or engine output name that will be connected to toField.

disconnect(...)

Disconnects any incoming connection from a field.

Args:

  • field: Field name of field to be disconnected.

enable_notify(...)

Enables or disables change notifications for a scene object.

Args:

    Boolean value indicating if notifications are enabled (True) or not (False).

get(...)

Returns a field or part by name.

Args:

  • name: Field or part name to be returned.
  • createIfNeeded: For node kit parts the second parameter controls if the named part should be created if is doesn't exist yet.

Returns:

    Field or node kit part if name is given. If no name is passed all field values are returned as string.

getfields(...)

Return the names and types of this container's fields.

Returns:

    List of tuples with name and type for each field of this field container instance.

getname(...)

Returns the instance name of a scene object.

Returns:

    String containing scene object name.

internal_pointer(...)

Return the internal field container pointer.

Returns:

    Internal pointer to field container instance.

isconnected(...)

Returns if a field has an incomming connection.

Args:

  • field: Field name whose conntection state will be checked.

Returns:

    True is the field field has an incoming connection from another field or engine output.

set(...)

Initializes fields or parts of a node kit.

Args:

    Initialization string containing field names and values.

setname(...)

Sets the instance name of a scene object.

Args:

    Name for scene object instance.

sotype(...)

Return the type name of a scene object.

Returns:

    String containing scene object type.

touch(...)

Marks a scene object as modified.
